# Real-run test pass — 2026-06-24

First exercise of the promptsmith engine on a live, unscripted user request (not a
crafted example). Goal: find friction the scaffold can't surface on its own.

## Input

`/promptsmith:sharpen` invoked on a real request:

> BoardPath (boardpath.tech and all subpages and routes) needs a UI analysis and bug
> cleanup. There are hidden buttons washed out by dark backgrounds, dead links, and areas
> that look unprofessional and unpolished.

A messy, multi-symptom, partly-subjective request — the kind the engine exists for.

## What the engine did

- **Routed** correctly to SHARPEN.
- **Extracted** goal, audience, stack, three named symptom classes.
- **Gap-filled** five labeled, reversible assumptions (repo path, defect-fix scope,
  auto-picked lenses, source access, git workflow).
- **Red-teamed** the request — surfaced the two real weaknesses: "looks unprofessional" is
  the vaguest input and needs a concrete standard; the contrast bug may be a theme-token
  problem, not 30 individual button fixes (wrong-layer trap).
- **Lens pass**: auto-picked visual-design, accessibility, ux-designer.
- **Synthesized** a complete ROLE/OBJECTIVE/CONTEXT/REQUIREMENTS/GUARDRAILS/SUCCESS/
  OUTPUT/OUT-OF-SCOPE prompt.
- **Surfaced** assumptions + offered `--deep`.

## --deep interview

Three open questions put to the user, all three answered, all three propagated correctly
into the final prompt — including a **scope flip** (defect-fix → redesign authorized) that
the synthesis correctly pushed into the guardrails (dropped "do not redesign," added
"converge on brand.md"). The pipeline handled a material change of intent without breaking.

## Findings

- **Engine logic: no defect.** Held on real, messy input. Route → extract → gap-fill →
  red-team → lens → synthesize → offer-depth all fired as specified. The `--deep` second
  pass correctly re-ran synthesis with real answers replacing assumptions.
- **Templates: no defect.** Every section filled; output read like an intentful prompt,
  not a form.
- **Bug found (separate thread): slash command registration.** The natural retry
  `/sharpen --deep ...` failed with `Unknown command: /sharpen. Did you mean /share?`.
  Only the namespaced `/promptsmith:sharpen` resolves. A user who learns `/sharpen` from
  the README will hit a dead end on the bare form. Tracked + fixed in a follow-up commit.

## Verdict

Engine and templates validated against real input. One install/registration defect found
and routed to its own fix. No changes to the engine warranted by this pass.
